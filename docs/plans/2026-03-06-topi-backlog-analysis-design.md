# Topi 待开发功能分析

> 分析日期：2026-03-06

## 1. 项目现状

Topi 是待办任务管理应用，支持 Web 界面与 agent MCP 集成。技术栈：前端 React Router + Vite + shadcn/ui，后端 Go + Gin + MySQL，MCP 暴露任务与列表工具。

### 1.1 已实现功能

| 模块 | 能力 |
|------|------|
| 任务 | 创建、更新、完成、放弃、恢复、删除、批量创建、拖拽排序 |
| 视图 | 收件箱、今日、明天、近七日、自定义列表、已完成、已放弃、回收站 |
| 字段 | 标题、富文本详情(TipTap)、优先级、截止日期、owner、estimatedHours |
| 认证 | 登录、注册、JWT |
| 集成 | agent MCP（SSE）、REST API、Swagger |
| 部署 | Docker、GitHub Actions 自动部署 |

### 1.2 待完善或待开发

- **乐观更新**：有设计文档，编辑任务名称会闪回旧值，未实现。
- **Dashboard**：仅展示「所有任务」，缺少统计/概览。
- **Docs**：占位页，用途未明确。

## 2. 设想待开发功能

基于项目结构与同类产品（Linear、Todoist、Things 等），提出以下待开发项，已录入 Topi 列表「Topi 待开发功能」：

| 功能 | 预估 | 说明 |
|------|------|------|
| 任务编辑乐观更新 | 1h | use-tasks 中先 setTasks 再 await，失败回滚 |
| 前端 owner 筛选 | 2h | API 已支持，前端加「我创建的 / Agent 创建的」 |
| Docs 页面定位 | 2h | 明确 docs 用途（帮助/文档视图）后实现 |
| Dashboard 增强 | 3h | 统计卡片、周报、完成率等 |
| 任务搜索 | 4h | 按标题/详情搜索，API 支持模糊查询 |
| 任务模板 | 4h | 保存常用任务为模板，快速创建 |
| 快捷入口 (Cmd+K) | 5h | 全局命令面板：创建、切换、搜索 |
| 标签系统 | 6h | 多标签、按标签筛选 |
| 日历视图 | 6h | 月/周视图、拖拽调整日期 |
| 提醒 / 通知 | 6h | 到期前浏览器/邮件通知 |
| OAuth 登录 | 8h | GitHub、Google 等 |
| 子任务 / 任务分解 | 8h | parent_id、层级展示、批量完成 |

## 3. 优先级建议

- **快速见效**：乐观更新、owner 筛选、Docs 定位
- **体验提升**：任务搜索、快捷入口、Dashboard
- **功能扩展**：标签、子任务、日历、模板、通知
- ** infra 增强**：OAuth
